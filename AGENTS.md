- 단, 사용자가 특정 작업군에 대해 "알아서 커밋/푸시"를 명시적으로 허용한 경우에는 해당 작업군의 후속 변경을 검증 완료 후 별도 재확인 없이 커밋/푸시한다.

- OneDrive 쪽은 회차/부별 원본 관리 구조라 파일명이 로컬과 다르다. 수정 전 반드시 대응되는 최종 코드 파일을 확인하고, 중복 백업 파일이나 `-DESKTOP-N9UCJ8E` 같은 임시/충돌 파일은 사용자가 요청하지 않는 한 수정하지 않는다.
- 대응 파일이 불명확하거나 OneDrive 경로가 없으면 임의로 덮어쓰지 말고 사용자에게 확인한다.

## 작업 종료 규칙

사용자가 "작업 종료 정리해 주세요"라고 말하면 다음을 수행한다.

1. 현재 Git 상태를 확인한다.
2. 변경 파일과 변경 이유를 정리한다.
3. `WORKLOG.md`를 이어 작업용으로 갱신한다.
4. 테스트 또는 검증 실행 여부를 기록한다.
5. 남은 작업과 주의사항을 기록한다.
6. `mock-exam` 또는 `quiz-1400` 코드 변경이 있었다면 OneDrive 대응 파일에도 동일 변경이 반영됐는지 확인한다.
7. 사용자가 명시적으로 허용한 경우에만 커밋/푸시한다.
