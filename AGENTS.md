- 작업 시작 시 WORKLOG.md를 먼저 읽는다.
- 작업 종료 전 WORKLOG.md를 갱신한다.
- 대화 전체가 아니라 이어 작업에 필요한 요약만 남긴다.
- 민감 정보, API 키, 토큰은 기록하지 않는다.
- 커밋/푸시는 사용자가 명시적으로 요청했을 때만 한다.

## 작업 종료 규칙

사용자가 "작업 종료 정리해 주세요"라고 말하면 다음을 수행한다.

1. 현재 Git 상태를 확인한다.
2. 변경 파일과 변경 이유를 정리한다.
3. `WORKLOG.md`를 이어 작업용으로 갱신한다.
4. 테스트 또는 검증 실행 여부를 기록한다.
5. 남은 작업과 주의사항을 기록한다.
6. 사용자가 명시적으로 허용한 경우에만 커밋/푸시한다.
