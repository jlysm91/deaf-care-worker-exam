- 작업 시작 시 WORKLOG.md를 먼저 읽는다.
- 작업 종료 전 WORKLOG.md를 갱신한다.
- 대화 전체가 아니라 이어 작업에 필요한 요약만 남긴다.
- 민감 정보, API 키, 토큰은 기록하지 않는다.
- 2026-06-11부터 이 저장소에서 Codex가 코드, 문서, 도구 파일을 수정한 경우 사용자가 매번 말하지 않아도 작업 완료 후 반드시 커밋/푸시한다.
- 기본 종료 흐름은 관련 검증 실행, OneDrive 대응 파일 동기화 확인, `WORKLOG.md` 갱신, `git status` 확인, 변경분 스테이징, 커밋, `origin/main` 푸시 순서다.
- `mock-exam` 또는 `quiz-1400` 코드 파일을 수정한 경우에는 사용자가 매번 말하지 않아도 반드시 OneDrive 우선 확인 흐름을 따른다. 순서는 로컬 파일 수정, 대응 OneDrive 최종 코드 파일 동기화/저장, OneDrive 또는 로컬 미리보기 기준 확인, 검증 실행, `WORKLOG.md` 갱신, Git 커밋, GitHub `origin/main` 푸시다.
- 위 OneDrive 우선 확인 흐름과 자동 커밋/푸시 규칙은 이 저장소의 고정 작업 규칙이다. 사용자가 명시적으로 "이 규칙을 바꿔라"라고 요청하지 않는 한 임의로 빼거나, 약화하거나, 삭제하지 않는다.
- 모의고사, 1400제, 실전 모의시험의 로그인 코드를 수정한 경우에는 OneDrive 최종 코드 파일 저장 후 올리사이트 관리자 홈페이지에도 최신 로그인 코드를 반영한다.
  - 올리사이트 반영은 사용자가 별도로 말하지 않아도 로그인 코드 수정 작업의 기본 완료 조건에 포함한다.
  - 반영 대상과 순서는 OneDrive 원본 관리 폴더의 `05. 올리사이트 코드 반영 작업표.xlsx`와 사용자가 입력한 올리사이트 관리자 위치 정보를 기준으로 확인한다.
  - 올리사이트 접속 아이디, 비밀번호, 세션 정보, API 키, 토큰은 `WORKLOG.md`, 커밋 메시지, 최종 답변에 기록하지 않는다.
  - 로그인 세션 만료, 권한 부족, 위치 정보 누락, 사이트 오류 등으로 직접 반영할 수 없으면 임의로 건너뛰지 말고 차단 사유와 미반영 대상을 보고한다.
- 사용자가 해당 작업에서 "커밋하지 마", "푸시하지 마", "검토만 해"처럼 명시적으로 금지한 경우에만 자동 커밋/푸시를 하지 않는다.
- 검증 실패, 원격 선행 커밋, 병합 충돌, 인증 문제처럼 바로 푸시할 수 없는 상황이 생기면 가능한 범위에서 해결을 시도하고, 해결 불가능한 차단 사유만 명확히 보고한다.
- 실제 작업 대상은 `02. 작업 목록` 아래의 파일이다.
- `02. 작업 목록/mock-exam` 또는 `02. 작업 목록/quiz-1400`의 코드 파일을 수정할 때는 아래 OneDrive 원본 관리 폴더의 대응 파일에도 동일한 변경을 반영한다.
  - 기준 경로: `C:\Users\COM-JHYUTG\OneDrive\바탕 화면\01. 개인용\01. 자기계발\01. 코딩\00. 업무 관련 코딩\1. 청각장애인 요양보호사 양성`
  - 모의고사 대응 폴더: `01. 합격!! 모의고사`
  - 1400제 대응 폴더: `02. 합격!! 1400題`
- OneDrive 쪽은 회차/부별 원본 관리 구조라 파일명이 로컬과 다르다. 수정 전 반드시 대응되는 최종 코드 파일을 확인하고, 중복 백업 파일이나 `-DESKTOP-N9UCJ8E` 같은 임시/충돌 파일은 사용자가 요청하지 않는 한 수정하지 않는다.
- 대응 파일이 불명확하거나 OneDrive 경로가 없으면 임의로 덮어쓰지 말고 사용자에게 확인한다.

## 작업 종료 규칙

사용자가 "작업 종료 정리해 주세요"라고 말하면 다음을 수행한다.

1. 현재 Git 상태를 확인한다.
2. 변경 파일과 변경 이유를 정리한다.
3. `WORKLOG.md`를 이어 작업용으로 갱신한다.
4. 테스트 또는 검증 실행 여부를 기록한다.
5. 남은 작업과 주의사항을 기록한다.
6. `mock-exam` 또는 `quiz-1400` 코드 변경이 있었다면 OneDrive 대응 파일에도 동일 변경이 반영됐는지 확인한다.
7. 사용자가 명시적으로 금지하지 않았다면 변경분을 커밋/푸시한다.
