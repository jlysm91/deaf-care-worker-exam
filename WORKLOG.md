# WORKLOG

## 1. 현재 프로젝트 목표

- 청각장애인 요양보호사 자격시험 대비 모의고사 웹 파일을 관리한다.
- A컴퓨터와 B컴퓨터에서 같은 GitHub 저장소를 기준으로 이어 작업할 수 있게 한다.
- 핵심 작업 대상은 `mock-exam`, `quiz-1400`, `vocabulary` 폴더이며, 문제/정답 데이터는 원본 PDF, 정답지, 검수 엑셀 기준으로만 수정한다.

## 2. 지금까지 완료한 작업

- `mock-exam` 1~10회 데스크톱용/모바일용 파일을 Git 저장소에 정리했다.
- 1회 모의고사에서 만든 모바일/태블릿 UI 개선 사항을 2~10회 모의고사에도 동일하게 반영했다.
- 스마트폰에서 내용이 넘칠 때만 문항 영역이 세로 스크롤되도록 조정했다.
- 태블릿 세로 화면에서는 남는 공간이 과도하지 않도록 문제 영역 크기를 자동 조절하도록 수정했다.
- 결과 페이지의 틀린 문항, 미응답 문항, 정답 문항 칩 목록을 화면 크기에 맞게 반응형으로 정리했다.
- 통합 엑셀과 PDF/정답지 기준으로 1~10회 총 800문항의 정답번호를 재검토했다.
- 엑셀 기준으로 발견된 정답번호 불일치 43건을 데스크톱용과 모바일용 모두 수정했다.
- 명확한 문구 오타 일부를 원문 확인 후 수정했다.
  - 예: `허루` -> `하루`
  - 예: `뇌졸중으로 뇌가 손상` -> `뇌졸중으로 소뇌가 손상`
  - 예: `영양주머니를 영양액을` -> `영양주머니에 영양액을`
  - 예: 3회 23번 중복 보기 수정
- `quiz-1400/login.html`을 추가했고, `quiz-1400/mobile-01.html`의 모바일 스크롤 관련 수정도 반영했다.

## 4. 실행한 테스트 또는 확인한 내용

- Git 상태 확인:
  - 브랜치: `main`
  - 원격 저장소: `origin` -> `https://github.com/jlysm91/deaf-care-worker-exam.git`
  - `HEAD`와 `origin/main` 동기화 여부 확인
- `mock-exam` 전체 데이터 검증:
  - 데스크톱 10개 파일, 모바일 10개 파일 모두 각 80문항인지 확인

  - 통합 엑셀 기준 800문항 정답번호 불일치가 0건인지 확인
- `git diff --check`로 공백 오류 및 기본 diff 오류 여부를 확인했다.
- 사용자가 모바일/태블릿 UI 적용 상태를 실제 화면에서 확인했다.

## 5. 아직 남은 작업

- 새 오타 검수표나 원본 PDF 기준 추가 수정 요청이 있으면 문제/보기/정답만 좁게 수정한다.
- 실제 GitHub Pages 또는 올리사이트 반영 화면에서 캐시가 남아 있을 수 있으므로 필요 시 새로고침 후 확인한다.
- `quiz-1400`의 나머지 회차나 로그인 흐름을 추가로 손볼 경우, `mock-exam`과 섞지 말고 별도 범위로 작업한다.
- 새 컴퓨터에서 이어 작업할 때는 먼저 원격 최신 상태를 가져와 로컬 상태를 맞춘다.

## 6. 다음 컴퓨터에서 이어서 작업할 때 주의할 점

- 작업 시작 전 반드시 실행한다.
  - `git status -sb`
  - `git fetch origin main`
  - `git rev-list --left-right --count HEAD...origin/main`
- 로컬이 뒤처져 있으면 먼저 `git pull --ff-only origin main`으로 최신화한다.
- 작업 중 로컬 변경이 있으면 덮어쓰지 말고 먼저 변경 파일을 확인한다.
- 문제/정답 데이터는 임의로 바꾸지 말고 원본 PDF, 정답지, 검수 엑셀 중 확인 가능한 기준이 있을 때만 수정한다.
- `videoId`는 문제 데이터와 연결되어 있으므로 문제/보기/정답 수정 중에도 함부로 삭제하거나 바꾸지 않는다.
- 데스크톱용과 모바일용은 같은 회차의 문제 데이터가 일치해야 한다.
- GitHub Pages나 외부 사이트는 반영 지연 또는 브라우저 캐시가 있을 수 있다.
- 민감 정보, 로그인 정보, API 키는 이 파일이나 코드에 기록하지 않는다.
