## 5. 아직 남은 작업

- 새 오타 검수표나 원본 PDF 기준 추가 수정 요청이 있으면 문제/보기/정답만 좁게 수정한다.
- 실제 GitHub Pages 또는 올리사이트 반영 화면에서 캐시가 남아 있을 수 있으므로 필요 시 새로고침 후 확인한다.
- `quiz-1400`의 나머지 회차나 로그인 흐름을 추가로 손볼 경우, `mock-exam`과 섞지 말고 별도 범위로 작업한다.
- 새 컴퓨터에서 이어 작업할 때는 먼저 원격 최신 상태를 가져와 로컬 상태를 맞춘다.
- 현재 `01. 청각장애인 요양보호사`는 빈 폴더라 Git에는 표시되지 않을 수 있다. 필요하면 실제 목록 파일을 추가한다.
- `mock-exam`, `quiz-1400` 수정 시 OneDrive 쪽 대응 파일 구조가 로컬과 다르므로 최종 코드 파일을 먼저 확인한다. 중복 백업 파일이나 `-DESKTOP-N9UCJ8E` 같은 충돌 파일은 사용자가 요청하지 않는 한 수정하지 않는다.
- OneDrive 1회 모의고사 최종 코드 폴더는 주력 3개 파일과 대비용 3개 파일, 총 6개 파일을 유지한다. 이후 1회 모의고사 수정 시 주력 데스크톱/모바일 파일을 우선 반영하고, 필요 시 대비용 파일도 최신 기준으로 맞춘다.
- 엑셀 텍스트와 HTML 텍스트를 완전히 맞추려면 먼저 엑셀 원본의 의심 행을 원본 PDF/docx로 재확인해야 한다. 예: 3회 10번 보기처럼 문항 주제와 맞지 않는 보기 텍스트가 `정상` 상태로 남아 있는 행이 있다.
